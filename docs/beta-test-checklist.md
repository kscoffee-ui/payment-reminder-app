# カイシュル βテストチェックリスト

## 1. βテストの目的

- イベント作成から参加者登録、支払い報告、幹事確認、LINE催促までの流れが問題なく動くか確認する
- スマホで迷わず操作できるか確認する
- 未払い者がすぐ分かるか確認する
- 本番URLで最低限安心して使える状態か確認する

## 2. テスト前提

- Firebase Hosting の本番URLで確認する
- スマホ表示を中心に確認する
- テスト用イベントを作成する
- 大量リロードや大量タブは避ける
- テスト後、不要な参加者やイベントは必要に応じて削除する

## 3. 幹事側テスト

- [ ] トップ画面を開ける
- [ ] イベントを作成できる
- [ ] 作成完了画面に参加者URLと幹事URLが表示される
- [ ] 管理画面に入れる
- [ ] ダッシュボードが崩れていない
- [ ] イベント概要カードの「>」から設定タブの編集画面へ移動できる
- [ ] 支払い状況サマリーが正しく表示される
- [ ] 未払い者カードの「すべて見る」から参加者一覧タブの未払いフィルターへ移動できる
- [ ] LINEで催促ボタンが押せる
- [ ] 設定タブでイベント情報を編集できる

## 4. 参加者側テスト

- [ ] 参加者URLを開ける
- [ ] 名前を入力して参加できる
- [ ] 支払い情報が表示される
- [ ] 支払い報告できる
- [ ] 報告後に確認待ち画面になる
- [ ] 幹事が確認済みにした後、確認済み画面になる
- [ ] 「この部屋から抜ける」が動く

## 5. 参加者一覧テスト

- [ ] すべて / 未払い / 報告済み / 確認済み のフィルターが動く
- [ ] 名前検索が動く
- [ ] 参加者詳細が確認できる
- [ ] 報告済みを確認済みにできる
- [ ] 参加者を削除できる
- [ ] 未払い者が多い場合でも一覧が見やすい

## 6. ダッシュボード表示テスト

- [ ] 未払い人数が正しく表示される
- [ ] 確認済み人数が正しく表示される
- [ ] 支払い完了率が正しく表示される
- [ ] 進捗バーが正しく表示される
- [ ] 未払い者プレビューは最大3人だけ表示される
- [ ] 4人以上いる場合は縦三点リーダ「︙」が表示される
- [ ] 「すべて見る」が青いリンク風に見える
- [ ] LINE催促ボタンが見本に近い見た目になっている

## 7. セルフテスト用パターン

- [ ] 参加者0人
- [ ] 参加者1人・未払い
- [ ] 参加者3人・全員未払い
- [ ] 参加者5人・未払い4人以上
- [ ] 報告済みがいる状態
- [ ] 確認済みがいる状態
- [ ] 全員確認済み
- [ ] 参加者を削除した後
