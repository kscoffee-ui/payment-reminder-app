# settings-ui-spec

## 参照画像

- 見本画像: `docs/wireframes/settings-ui-reference.png`
- 共通設計: `docs/設計図.txt`
- 旧UI思想: `docs/フレームワーク.jpeg`

## この画面の目的

幹事がイベント情報を確認・編集し、参加者追加用の共有導線をすぐ使えるようにする。

この画面では、以下を最優先する。

- イベント情報をカードで見やすく整理する
- 編集導線を分かりやすくする
- 参加者追加の共有導線を目立たせる
- LINE共有を迷わず押せるようにする
- 設定画面を複雑にしすぎない

## 画面名

設定 / イベント情報

## 対象画面

幹事側管理画面の設定タブ。

想定URL:

```txt
/admin/:eventId?token=:adminToken
UI全体方針

見本画像 settings-ui-reference.png にできるだけ近づける。

モバイルファースト
背景は薄いグレー
カードは白
情報をカード単位で整理する
アイコンは青を基調にする
参加者追加は緑系で強調する
角丸は大きめ
影は薄く自然に
アイコンは lucide-react を使用する
PNG/JPGアイコンは追加しない
画面構成
1. 上部ヘッダー

表示内容:

中央にカイシュルロゴ
右上にベルアイコン風の装飾を置いてもよい

注意:

通知機能は実装しない
ベルアイコンは装飾扱い
クリック処理は追加しない
2. ページタイトル

表示内容:

設定

その下にセクション見出し:

イベント情報
3. イベント情報2×2カード

見本画像のように、イベント情報を2×2のカードで表示する。

表示カード:

イベント名
アイコン: FileText / Notebook系
ラベル: イベント名
値: event.title
日付
アイコン: Calendar
ラベル: 日付
値: event.eventDate
金額
アイコン: JapaneseYen
ラベル: 金額
値: ¥3,000 のように表示
支払い方法
アイコン: Wallet
ラベル: 支払い方法
値: 支払い方法の日本語表記

支払い方法の表示例:

cash -> 現金回収
paypay -> PayPay
bank -> 銀行振込
other -> その他

注意:

Firestoreの値は変更しない
表示だけ日本語に整える
カード内の情報は短く見やすくする
4. 幹事からの案内カード

表示内容:

アイコン: Megaphone
ラベル: 幹事からの案内
本文: event.paymentInfo または event.memo

優先表示:

paymentInfo
memo
何もない場合は空状態テキスト

空状態例:

案内はまだ設定されていません

注意:

支払い情報は参加者にとって重要なため、省略しすぎない
長すぎる場合は折り返して表示する
ただしカードが巨大になりすぎないようにする
5. イベント情報編集ボタン

見本画像のようにアウトラインの大きなボタンを表示する。

表示文言:

イベント情報を編集

アイコン:

Pencil

仕様:

押すと既存の編集モードに切り替える
既存の設定タブ編集機能を維持する
新しい独立編集画面は作らなくてよい

注意:

保存前確認ダイアログは維持する
バリデーションは既存仕様を維持する
保存中は二重送信防止を維持する
6. 参加者管理セクション

セクション見出し:

参加者管理
7. 参加者を追加するカード

見本画像のように、参加者追加の共有導線をカードで表示する。

表示内容:

アイコン: UserPlus
タイトル: 参加者を追加する
説明文:
LINEグループに送ると、参加者が自分で名前を入力して参加できます。

ボタン:

LINEで共有
その他のアプリで共有

デザイン:

LINEで共有: 緑系
その他のアプリで共有: 白または薄いグレーのアウトライン
2つのボタンは横並び
スマホ幅で詰まる場合は縦並びにしてよい

使用ロジック:

LINE共有は既存の共有処理を使う
その他のアプリで共有は既存の Web Share API 処理を使う

注意:

参加者用URLの全文表示はしない
幹事用URLの全文表示はしない
メッセージコピー機能は追加しない
参加者はURLを開いて自分で名前を入力する仕様を維持する
8. 参加者一覧を見るカード

見本画像に合わせて、参加者一覧への導線を置いてもよい。

表示内容:

参加者一覧を見る

仕様:

押すと参加者一覧タブへ切り替える
新しいページは作らなくてよい
9. その他セクション

見本画像には以下のような項目がある。

使い方ガイド
よくある質問

MVPでは必須ではない。

実装する場合:

見た目だけの静的項目として追加してよい
ただし未実装のページへ遷移させない
クリック不可、または後で実装予定の見た目に留める

推奨:

最初の実装では追加しなくてもよい
ワイヤー完全一致を優先する場合のみ、装飾的に表示する
10. 編集モード

既存の編集モードを維持する。

編集対象:

title
eventDate
amountPerPerson
paymentMethod
paymentInfo
memo

バリデーション:

イベント名必須
日付必須
1人あたりの金額は1円以上
支払い方法必須
支払い情報必須
メモ任意

保存時:

保存前に確認ダイアログを表示する
保存中はボタンをdisabledにする
成功時は通常表示モードに戻す
失敗時は入力内容を保持してエラー表示する
11. ボトムタブ

既存のボトムタブを維持する。

表示項目:

ダッシュボード
参加者一覧
設定

この画面では 設定 を active にする。

禁止事項

今回のUI調整では以下を変更しない。

Firestore構造
URL設計
token検証
localStorage仕様
kaishuruAdminEvents の保存仕様
参加者自己登録フロー
支払い報告ロジック
幹事確認ロジック
参加者削除ロジック
LINE共有処理の中身
Web Share API処理の中身
参加者側画面
決済機能なしの方針
メッセージコピー機能なしの方針
通知機能
実装時の注意
既存の設定タブ編集機能を壊さない
参加者用URLと幹事用URLの扱いを間違えない
参加者用URLは共有導線に使う
幹事用URLは参加者に共有しない
URL全文表示は復活させない
カードの余白は見本画像に寄せる
npm run lint が通ること
npm run build が通ること
完了条件
見本画像に近い設定UIになっている
イベント情報が2×2カードで表示される
幹事からの案内カードが表示される
イベント情報編集ボタンが動く
編集・保存・キャンセルが既存通り動く
参加者追加カードが表示される
LINE共有が既存通り動く
その他のアプリで共有が既存通り動く
参加者一覧への導線がある
ダッシュボード・参加者一覧・参加者側画面に影響がない
