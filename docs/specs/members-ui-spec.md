# members-ui-spec

## 参照画像

- 見本画像: `docs/wireframes/members-ui-reference.png`
- 共通設計: `docs/設計図.txt`
- 旧UI思想: `docs/フレームワーク.jpeg`

## この画面の目的

幹事が参加者ごとの支払い状態を素早く確認し、未払い・確認待ち・確認済みを絞り込みやすくする。

40人規模のサークル・飲み会でも一覧性を損なわないことを重視する。

この画面では、以下を最優先する。

- 未払い者をすぐ発見できる
- 状態が色付きバッジで一目で分かる
- 大人数でもスクロールしやすい
- 報告済み / 確認待ちの参加者を見つけやすい
- 不要な縦幅を減らす

## 画面名

参加者一覧

## 対象画面

幹事側管理画面の参加者一覧タブ。

想定URL:

```txt
/admin/:eventId?token=:adminToken
UI全体方針

見本画像 members-ui-reference.png にできるだけ近づける。

モバイルファースト
背景は薄いグレー
カードは白
1人1人の表示はコンパクトにする
状態バッジで支払い状況を明確にする
40人規模でも見やすい行間にする
アイコンは lucide-react を使用する
PNG/JPGアイコンは追加しない
画面構成
1. 上部ヘッダー

表示内容:

中央にカイシュルロゴ
右上にベルアイコン風の装飾を置いてもよい

注意:

通知機能は実装しない
ベルアイコンは装飾扱い
クリック処理は追加しない
2. ページタイトル

表示内容:

参加者一覧

タイトルは左寄せで表示する。

3. 検索バー

見本画像のように、タイトル下に検索バーを置く。

表示内容:

プレースホルダー: 名前で検索
左または右に検索アイコン

検索対象:

参加者名

仕様:

入力した文字を含む参加者だけを表示する
大文字小文字は日本語中心なので深く考えなくてよい
空文字の場合は全員表示する

注意:

Firestore検索ではなく、取得済み members 配列に対するフロント側フィルターでよい
Firestore構造は変更しない
4. ステータスフィルター

検索バーの下にステータスフィルターを配置する。

表示項目:

すべて
未払い
報告済み
確認済み

表示例:

[すべて] [未払い] [報告済み] [確認済み]

ステータス色:

すべて: 青
未払い: 赤
報告済み: オレンジ
確認済み: 緑

仕様:

選択中のフィルターは塗りまたは強い枠線で強調する
クリックで表示対象を切り替える
検索文字列とステータスフィルターは同時に効く

初期表示:

既存実装に合わせて すべて でもよい
未払い回収の思想を強める場合は 未払い 初期表示も検討可
ただし今回の見本一致では すべて 初期表示を優先してよい
5. 参加者リスト

見本画像のようなコンパクトな1行リストにする。

各行の表示内容:

左に丸いイニシャルアイコン
参加者名
更新日時
ステータスバッジ
右端に chevron / 矢印

表示例:

田中 太郎
5/29 10:20 更新
[未払い] >

表示項目:

member.name
member.status
member.updatedAt
event.amountPerPerson
member.paymentMethod
member.proofMemo

ただし、一覧の基本表示では以下を優先する。

名前
更新日時
ステータス

金額や支払い方法を表示する場合は、行が高くなりすぎないよう注意する。

6. ステータス表示

ステータスは必ず色付きバッジで表示する。

unpaid

表示文言:

未払い

色:

赤系
背景は薄い赤
文字は濃い赤
reported

表示文言:

報告済み

または

確認待ち

色:

オレンジ系
背景は薄いオレンジ
文字は濃いオレンジ
confirmed

表示文言:

確認済み

色:

緑系
背景は薄い緑
文字は濃い緑
7. 報告済み参加者の操作

参加者一覧では、報告済み / 確認待ちの参加者に 確認済みにする 操作を表示しない。

表示内容:

- 名前
- 更新日時
- 確認待ちステータスバッジ
- 詳細領域の支払い情報

支払い確認の操作は、ダッシュボード右上のベルから開く確認待ち一覧に集約する。
確認待ち一覧では、既存どおり 確認済みにする と 未払いに戻す を利用できる。

注意:

参加者一覧では reported / unpaid / confirmed のいずれにも確認済みにするボタンを出さない
既存の幹事確認ロジックは変更しない
確認待ち一覧の確認済みにする / 未払いに戻す 導線は維持する
8. 削除操作

参加者削除機能は既存仕様として残す。

ただし、一覧を圧迫しないようにする。

推奨表示:

小さいゴミ箱アイコン
または行を展開したときだけ削除ボタンを表示

注意:

削除確認ダイアログは維持する
削除ロジックは変更しない
削除ボタンを大きくしすぎない
9. 空状態

参加者が0人の場合:

まだ参加者はいません
参加者URLを共有すると、参加者が自分で名前を入力して参加できます。

検索結果が0件の場合:

該当する参加者はいません
10. ボトムタブ

既存のボトムタブを維持する。

表示項目:

ダッシュボード
参加者一覧
設定

この画面では 参加者一覧 を active にする。

禁止事項

今回のUI調整では以下を変更しない。

Firestore構造
URL設計
token検証
localStorage仕様
参加者自己登録フロー
支払い報告ロジック
幹事確認ロジック
参加者削除ロジック
LINE共有処理
Web Share API処理
設定タブの編集機能
参加者側画面
決済機能なしの方針
メッセージコピー機能なしの方針
通知機能
実装時の注意
40人規模でも見やすいように縦幅を抑える
1行あたりの高さは大きくしすぎない
ただしタップしにくくならないよう最低限の高さを確保する
ステータス色はダッシュボードと統一する
スマホ幅で横スクロールが出ないようにする
npm run lint が通ること
npm run build が通ること
完了条件
見本画像に近い参加者一覧UIになっている
検索バーがある
ステータスフィルターがある
参加者がコンパクトな行リストで表示される
未払い / 報告済み / 確認済みが色付きバッジで分かる
参加者一覧の確認待ちフィルターでは確認待ちステータスだけが表示され、確認済みにするボタンが出ない
ベルから開く確認待ち一覧では確認済みにする操作が既存どおり使える
参加者削除が既存通り動く
ダッシュボード・設定・参加者側画面に影響がない
