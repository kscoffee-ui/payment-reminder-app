# 未払い回収ツール 設計図（最新版）

## プロダクト定義
- 本アプリは「割り勘計算アプリ」ではなく「未払い回収ツール」。
- 幹事の心理的負担を下げ、未払い者の把握と催促を素早く行う。

## 基本方針
- 幹事はイベント情報のみを作成し、参加者名は事前入力しない。
- 参加者は参加者URLから自分で名前を入力して参加する。
- ログインは実装しない。トークン付きURLで最低限の制御を行う。

## 画面
### 幹事側
1. イベント作成画面
2. 作成完了画面
3. 管理画面（ダッシュボード）
4. 参加者一覧
5. 催促画面（LINE共有のみ、コピー機能なし）

### 参加者側
6. 参加登録画面
7. 支払い画面
8. 報告済み画面（確認待ち）
9. 確認済み画面
10. この部屋から抜ける（localStorage解除）

## ステータス
- unpaid（未払い）
- reported（報告済み / 確認待ち）
- confirmed（確認済み）

## Firestore
### events/{eventId}
{
  id: string,
  title: string,
  eventDate: string,
  amountPerPerson: number,
  paymentMethod: "paypay" | "cash" | "bank" | "other",
  paymentInfo: string,
  memo: string,
  participantToken: string,
  adminToken: string,
  createdAt: timestamp,
  updatedAt: timestamp
}

### events/{eventId}/members/{memberId}
{
  id: string,
  name: string,
  status: "unpaid" | "reported" | "confirmed",
  paymentMethod: "paypay" | "cash" | "bank" | "other",
  proofMemo: string,
  createdAt: timestamp,
  updatedAt: timestamp
}

## URL
- 参加者: /join/:eventId?token=:participantToken
- 幹事: /admin/:eventId?token=:adminToken

## localStorage
- キー: paymentReminderMember:{eventId}
- 値: { eventId, memberId, memberName }

## リアルタイム同期
- Firestore onSnapshot を使用する。
- 幹事側: event + members を購読。
- 参加者側: 自分のmemberを購読。

## 非対応（実装しない）
- イベントロック
- メッセージコピー
- 決済機能
- PayPay自動送金
- 送金完了自動検知
- ログイン機能
- 参加者側催促画面
